The pipeline

per plan · driven to done
↻

Start · validate + reconcile

validatesettles truth

First validate the config — /crosscut validate runs a whole-file check; errors or bad YAML halt before any plan is touched. Then reconcile: settle every plan's real status from the highest signal down — the newest completed run's head_sha reachable from the integration branch (covers a merge commit and a fast-forward), then run.json, then a live-process check — reclaim stale per-repo locks, and prune run records by retention.

01

Authoring

→ draft

Write the plan: frontmatter + code-first ### Task N sections. A non-trivial architecture call spins up a research subagent first.

02

Self-review

draft → todo

Up to five improve cycles — completeness, ordering, no placeholders — and enforce the product boundary on depends_on. Always runs, no external tool.

03

Plan review

Gate · the planplan_reviewtodo → validated

An independent read-only pass over the plan, before any code exists — codex, claude, or none. MUST-FIX findings loop back; the rest is applied.

04

Executor run

executorvalidated → running

Build the plan on an isolated <slug> worktree — ralphex (Docker), claude (in-session), or codex. A per-repo lock keeps one executor per repo; different repos run in parallel.

05

Acceptance · 5a tests + 5b final review

Gate · the codefinal_review→ accepted

5a runs lint + tests and reads the actual output. 5b — final review reviews the produced code/diff before merge — in-session, claude, codex, or none. A material finding is a blocker.

06

Finalize · merge

accepted → merging → done

Local merge (--no-ff by default), move the plan to completed/, set the ROADMAP to done, and prune the plan's run records. Push only when explicitly enabled.

07

Continue

max_parallelloop until backlog empty

Don't stop at one plan — drive every ready plan. Concurrent across repos, sequential within a repo; merges stay sequential per repo.

Two gates, two subjects

neither substitutes for the other
Phase 3

Plan review — the plan

Reviews the plan before it is built. Catches a bad approach while it is still cheap to change. Read-only; feeds an apply / re-review loop.

Phase 5b

Final review — the code

Reviews the produced code before it merges. The last gate; a different reviewer or model catches what the author rationalizes.

How it stays safe & autonomous

Asks only onArchitecture decisions and blockers — otherwise it runs to done without a go-ahead at each step.
IsolationEvery run lands on its own <slug> worktree; the branch is the only deliverable until Phase 6.
ParallelismParallel across repos, sequential within one — an atomic per-repo lock, optional max_parallel cap.
Git safetyNever pushes unless enabled · merges --no-ff · no Co-Authored-By trailer.

One config drives all of it

~/.crosscut/crosscut.config.yaml
crosscut.config.yaml
executor: ralphex          # ralphex · claude · codex
plan_review: codex         # reviews the PLAN  — codex · claude · none
final_review: in-session   # reviews the CODE  — in-session · claude · codex · none
git:
  merge_ff: false          # --no-ff merges
  push_enabled: false      # local-merge-only, the safe default
knowledge_base:
  path: ~/.crosscut/knowledge   # Obsidian-compatible
executor_options:
  runs_retention_days: 0    # prune a plan's runs once it is done
  model: inherit           # inherit · opus · sonnet · haiku · fable (claude) · or a codex model
  reasoning_effort: inherit # inherit · low · medium · high · xhigh · max
repos:
  - name: core            # grouped into products; boundaries enforced
    product: platform
